# DeepTicker — 주식 & ETF 분석기

ETF나 주식 이름을 전달받아 공식 자료를 확인한 뒤, 상세 분석을 정적 HTML 리포트로 누적하는 프로젝트입니다.
별도 서버나 설치 없이 `index.html`을 열어 볼 수 있고 GitHub Pages로 같은 화면을 공유할 수 있습니다.

## 현재 범위

- ETF명/종목코드 검색
- 국내상장 ETF와 해외상장 ETF 분리 조회
- ETF별 분석 날짜 아카이브(이전 분석 보존)
- 상품 구조, 비용, 지수 방법론, 주요 구성 종목
- 구성 비중 차트와 배당·분배금 상세 이력
- 정상 표시가 확인된 종목은 TradingView, 제한 종목은 저장된 1년 일봉 자체 차트
- 동종 ETF 보수·유동성·괴리·듀레이션·집중도 비교
- 직전 분석 대비 새 정보와 유지된 판단 요약
- 지수 성과와 변동성, 상승 동력, 위험, 시나리오
- 매수 전 체크리스트와 원문 출처
- 모바일 반응형 화면
- 주식별 사용자 평가와 Codex 독립 평가 분리
- 단기·중기·장기 중 추천 보유 관점과 5점 만점 최종 가중점수
- 주식별 주가 차트와 최근 5년 20개 분기 매출·영업이익 구조
- 근거 기반 Codex 전망, 밸류에이션, 촉매·위험, 날짜별 기록
- 기타 지표 탭: 고중요 경제 이벤트, 환율·크립토·원자재 TradingView 차트, 다음 3개 FOMC 확률 스냅샷
- 채팅 탭: 지인들이 익명 랜덤 이름으로 ETF·주식 분석 요청을 남기는 댓글함

현재 ETF는 `KODEX 차이나휴머노이드로봇(0048K0)`, `TIGER 미국배당다우존스(458730)`, 미국상장 `TLT`, 주식은 미국상장 `META`와 국내상장 `LG에너지솔루션(373220)` 리포트가 들어 있습니다.

## 사용 방법

`index.html`을 더블클릭하면 브라우저에서 바로 열립니다.

ETF 분석 데이터는 `data/etfs.js`의 `window.ETF_ANALYSES` 배열에 하나씩 추가합니다.
같은 ETF를 다시 조사할 때는 기존 값을 덮어쓰지 않고 해당 ETF의 `snapshots` 배열 맨 앞에 새 날짜 기록을 추가합니다.
가격, 보유 종목, 수익률처럼 바뀌는 숫자에는 반드시 기준일을 함께 기록합니다.

주식 분석 데이터는 `data/stocks.js`의 `window.STOCK_ANALYSES`에 저장합니다. ETF와 동일하게 같은 종목을 다시 조사할 때 기존 기록을 덮어쓰지 않고 `snapshots` 배열에 새 날짜 기록을 추가합니다.
사용자 평가는 1~8번 항목을 1~5점으로 기록하되 7·8번 항목은 2배 가중하고, 공유 화면에는 최종 가중평균 점수만 보여줍니다. 단기·중기·장기는 각각 평가하는 칸이 아니라 해당 종목을 어떤 보유 관점으로 볼지 추천하는 값으로 남깁니다. Codex 평가는 사용자 평가와 독립적으로 작성합니다.

기타 지표 데이터는 `data/indicators.js`의 `window.OTHER_INDICATORS`에 저장합니다. TradingView에서 표시 가능한 심볼은 페이지 안에서 바로 띄우고, 경제 캘린더와 FedWatch 확률은 업데이트 시점의 스냅샷으로 남깁니다.

채팅/분석 요청 댓글함은 `requests.js`와 `data/request-board-config.js`가 담당합니다. GitHub Pages는 정적 사이트라 방문자 댓글을 자체 저장할 수 없으므로 Supabase 같은 외부 DB 연결이 필요합니다. Supabase SQL 설정 예시는 `docs/supabase-analysis-requests.sql`에 있습니다. 연결 후에는 방문자가 이름 없이 요청을 남기고, 댓글마다 새 랜덤 익명명이 붙습니다. 삭제 권한은 Supabase RLS에서 지정한 관리자 이메일에만 부여합니다.

## 자동 업데이트

`.github/workflows/update-economic-calendar.yml`은 매주 월요일 오전 8:30(KST)에 GitHub Actions에서 실행됩니다. 이 작업은 Investing.com 경제 캘린더의 이번 주 중요도 3 이벤트만 받아 `data/indicators.js`의 경제 이벤트 영역을 갱신하고, 변경 사항이 있으면 자동 커밋·푸시합니다.

수동으로 즉시 실행하고 싶을 때는 GitHub 저장소의 `Actions` 탭에서 `Update weekly economic calendar` 워크플로우를 선택한 뒤 `Run workflow`를 누르면 됩니다.

## 데이터 원칙

1. 운용사 상품 페이지·투자설명서·지수사업자 방법론 등 1차 자료를 우선합니다.
2. 시세, 구성 종목, 성과는 기준일을 반드시 표시합니다.
3. 사실과 분석 의견을 구분합니다.
4. 특정 상품의 매수·매도 권유나 목표가격을 제시하지 않습니다.
5. 업데이트 때 이전 기록을 지우지 않고 새로운 날짜 스냅샷을 추가합니다.
